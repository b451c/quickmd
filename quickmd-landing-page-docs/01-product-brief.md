# QuickMD - Product Brief

## Quick Summary

QuickMD is a free, lightweight macOS app that lets you instantly preview Markdown files with beautiful rendering, syntax-highlighted code blocks, and full dark mode support - no editor required.

---

## Elevator Pitch

**One-liner:** "The fastest way to read Markdown on Mac."

**Extended (2 sentences):** QuickMD is a native macOS app that transforms how you view Markdown files. Double-click any .md file to instantly see beautifully rendered text, code with syntax highlighting, tables, and images - without opening a heavy editor.

---

## Problem Statement

### Pain Points QuickMD Solves:

| Problem | Current Solutions | QuickMD Solution |
|---------|-------------------|------------------|
| "I just want to READ this README" | Open VS Code, wait for it to load | Double-click, instant preview |
| "Markdown looks ugly in text editors" | Copy to web viewer, browser extensions | Native app, beautiful rendering |
| "No syntax highlighting in Quick Look" | Open full IDE just to see code | Built-in syntax highlighting |
| "Preview apps are bloated with editing" | Pay for Markdown editors | Free, view-only, lightweight |
| "Dark mode breaks in most viewers" | Manual theme switching | Auto dark/light mode |

### The "Aha" Moment:
You receive a README.md or documentation file. You don't want to edit it - you just want to READ it beautifully. QuickMD does exactly that.

## Target Audience

### Primary Users:

1. **Developers (70%)**
   - Read README files, documentation, changelogs
   - Review Markdown before committing
   - Quick preview without opening IDE
   - Age: 25-45, tech-savvy

2. **Technical Writers (15%)**
   - Proofread documentation
   - Review formatting before publishing
   - Check rendered output

3. **Students & Academics (10%)**
   - Read course materials in Markdown
   - View lecture notes
   - Research paper previews

4. **Knowledge Workers (5%)**
   - Read shared notes from developers
   - View exported documentation
   - Non-technical Markdown consumers

### User Personas:

**"Busy Dev Dana"**
- Senior developer, uses Mac
- Gets 10+ README files daily
- Hates waiting for VS Code to open
- Values: Speed, simplicity, keyboard shortcuts

**"Docs Derek"**
- Technical writer at SaaS company
- Writes in Markdown, needs quick previews
- Wants to see how docs will render
- Values: Accurate rendering, clean UI

## Unique Value Proposition (USP)

### What Makes QuickMD Different:

| Competitor | Their Approach | QuickMD Advantage |
|------------|----------------|-------------------|
| VS Code | Full IDE, editing-focused | View-only, instant launch |
| Typora | Paid ($15), WYSIWYG editor | Free, pure viewer |
| MacDown | Split-pane editor | Simpler, no editing complexity |
| Marked 2 | Paid ($14), many features | Free, focused on essentials |
| Quick Look | Basic, no syntax highlighting | Full rendering + highlighting |

### QuickMD's Unfair Advantages:
1. **Free** - No cost, no trial limits
2. **Native** - True macOS app, not Electron bloat
3. **Fast** - ~2MB, launches instantly
4. **Focused** - Does one thing perfectly
5. **Beautiful** - Proper syntax highlighting, dark mode
6. **Zero config** - Works out of the box

---

## Core Features (Priority Order)

### Must-Highlight Features:

1. **Instant Preview**
   - Double-click any .md file
   - No waiting, no loading screens
   - Native macOS file integration

2. **Syntax Highlighting**
   - 20+ languages supported
   - Keywords, strings, comments, numbers
   - Proper code block rendering

3. **Full Markdown Support**
   - Tables (GFM style)
   - Task lists with checkboxes
   - Images (local + remote)
   - All formatting (bold, italic, etc.)

4. **Dark Mode**
   - Automatic system detection
   - Beautiful in both modes
   - Optimized color schemes

5. **Lightweight**
   - ~2MB app size
   - Minimal memory usage
   - No bloat, no electron

## Brand Voice & Tone

### Personality:
- **Professional** but approachable
- **Technical** but not intimidating
- **Confident** but not arrogant
- **Minimal** - let the product speak

### Tone Guidelines:
- Use simple, direct language
- Avoid marketing fluff
- Focus on practical benefits
- Speak to developers as peers

### Do Say:
- "Instant preview"
- "No bloat"
- "Just works"
- "Free, forever"

### Don't Say:
- "Revolutionary" (overused)
- "Best-in-class" (vague)
- "Powerful" (means nothing)
- "Game-changing" (cringe)
